# 传统测试用例转BDD(Gherkin)标准规范手册（简化版）

**适配：Python + Playwright + Codegen录制 + AI自动生成脚本**

**核心：拆细测试点、固定数据、精准操作、明确断言，合理运用数据驱动，确保用例可直接落地**

## 一、核心目标

1. 用例可直接配合Playwright Codegen录制，无需补充操作细节
2. AI可基于用例生成可运行的自动化代码（含定位、操作、断言）
3. 消除模糊描述，每个测试点拆细至可验证
4. 运用数据驱动简化冗余场景，提升用例可读性和维护性

## 二、前置统一约定

### 2.1 禁用话术（绝对不允许出现）

- ❌ 功能正常、校验通过、符合要求、任意字符、随机填写、自行输入
- ❌ 筛选正常、排序正常、链接可跳转等模糊操作/断言
- ❌ 同类场景重复编写，不使用数据驱动

### 2.2 强制替换规则

- ✅ 所有输入：写固定实测文本/URL/符号，拒绝模糊表述
- ✅ 所有操作：写精准动作（输入+触发、点击次数、具体选项）
- ✅ 所有断言：写肉眼可见+代码可捕获的结果（提示、条数、状态等）
- ✅ 所有功能：拆「正向+反向+异常/恢复」测试点
- ✅ 同类场景：必须用数据驱动（Scenario Outline+Examples）批量覆盖

## 三、Gherkin关键字标准用法（适配录制+AI）

### 3.1 Feature

定义独立业务模块，明确角色+测试目的，一个模块一个Feature。

### 3.2 Background

收纳所有场景共用前置（登录、进入页面、预置数据），只执行1次，不写操作和断言。

### 3.3 Given

描述已有固定状态/预置数据，不写操作，数据必须固定可复用，数据驱动场景需关联Examples参数。

### 3.4 When（最关键）

写可录制的精准UI操作，步骤与实际操作一致，不跨步骤、不省略，数据驱动步骤需通用适配所有参数。

1. 输入/筛选：操作对象+固定内容+触发动作（回车/点击）
2. 排序：点击表头+点击次数
3. 按钮/下拉/弹窗：操作对象+动作+具体选项

### 3.5 Then

写可自动化校验的结果，每个When对应至少一个断言，与操作强关联，文案与前端一致。

1. 文本/数据断言：固定提示、数据展示
2. 状态断言：按钮灰化/可点击、排序箭头
3. 数据条数/跳转/恢复断言：条数、地址、原始状态恢复

### 3.6 And

接续同类型步骤，不跨类型混用，提升可读性。

### 3.7 Scenario Outline + Examples（数据驱动核心）

适用：多枚举、多状态组合、多字段校验、同类批量操作场景。

1. Examples表格填死固定数据，与Background预置数据对应
2. 步骤通用适配所有参数，参数命名清晰且与步骤一致
3. 枚举/按钮权限校验：明确数量、内容或状态组合，断言精准

## 四、测试数据编写规范（通用）

1. 名称类：中文+字母+数字+特殊符号（如：测试压缩工具001@#）
2. 版本类：标准固定版本号（如：V3.0.2-HF02）
3. URL类：同时提供合法+非法样例，覆盖正反校验
4. 枚举类：所有选项写进Examples，不漏项
5. 数据驱动：Examples数据与Background预置数据完全一致

## 五、场景拆分规范

1. 核心：一个Scenario只做「单一操作+对应断言」
2. 拆分要求：新增、校验、编辑、删除、并发等拆独立场景；复杂功能拆正反+恢复子场景；并发写清操作顺序；同类场景用数据驱动

## 六、高频场景核心要求（极简版）

1. 筛选：必拆正向+反向+清空筛选3个测试点
2. 排序：必测正向+反向双向，明确点击次数和箭头状态
3. 按钮：单个按钮独立场景，覆盖点击+结果+状态
4. 超链接：校验样式+跳转+跳转后状态（无报错）
5. 并发：写清A/B浏览器操作顺序，断言无脏数据
6. 枚举/按钮权限：必用数据驱动，批量覆盖所有选项/状态组合

## 七、常见踩点修正（避坑核心）

| 常见坑点 | 修正方案 |
|---------|---------|
| 筛选漏反向/清空、排序漏箭头 | 筛选拆3点，排序测双向+箭头 |
| 测试数据模糊、断言不具体 | 补固定数据，断言到具体文案/地址/状态 |
| 一个场景多操作、同类场景重复写 | 拆分场景，同类用数据驱动简化 |
| 并发操作顺序模糊 | 写死A/B浏览器操作顺序 |
| 数据驱动参数/数据不对应 | 参数命名清晰，与Background数据一致 |

## 八、极简落地口诀

1. 删模糊表述，全补固定数据
2. 筛选拆3点，排序测双向
3. 一场景一件事，同类场景用数据驱动
4. 操作精准写，断言具体写

本规范适配所有Web UI传统用例转BDD，核心是"拆细、固定、精准、驱动"，确保用例可直接配合Playwright录制、AI生成脚本，无需额外补充细节。
