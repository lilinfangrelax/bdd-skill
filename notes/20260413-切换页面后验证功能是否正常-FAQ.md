出现过的Bug：**切换页面后（从A页面切换到B页面），B页面上的功能出现异常**。
> 如何解决此类切换问题？BDD测试用例应该如何设计？

方式一：新增独立 Scenario
```gherkin
Feature: 订单页功能

  # 原有用例 —— 直接访问
  Scenario: 直接进入订单页，查询功能正常
    Given 用户直接打开"订单页"
    When 用户点击"查询"按钮
    Then 应返回订单列表数据

  # 新增用例 —— 从B页切换过来
  Scenario: 从"购物车页"切换回订单页后，查询功能仍正常
    Given 用户已在"订单页"完成一次查询
    And 用户切换至"购物车页"
    And 用户从"购物车页"返回"订单页"
    When 用户点击"查询"按钮
    Then 应返回订单列表数据
```
方式二：Scenario Outline（多个来源页面时）
```gherkin
Scenario Outline: 从其他页面切换回订单页后，查询功能仍正常
    Given 用户已在"订单页"完成一次查询
    And 用户切换至 "<来源页>"
    And 用户从 "<来源页>" 返回"订单页"
    When 用户点击"查询"按钮
    Then 应返回订单列表数据

    Examples:
      | 来源页   |
      | 购物车页 |
      | 商品详情页 |
      | 个人中心页 |
```
方式三：Background（整个 Feature 都需要"切换过来"的前提）
```gherkin
Feature: 从B页切换回来后的订单页功能验证

  Background:
    Given 用户已在"订单页"完成一次查询
    And 用户切换至"购物车页"
    And 用户从"购物车页"返回"订单页"

  Scenario: 查询功能正常
    When 用户点击"查询"按钮
    Then 应返回订单列表数据

  Scenario: 筛选功能正常
    When 用户选择筛选条件"待付款"
    Then 列表只显示状态为"待付款"的订单
```

*在featureA文件中，验证页面A的所有功能。然后再新建一个featureAB文件，加一个background，验证页面B切换到页面A的之后，页面A的所有功能*
```
features/
├── pageA.feature          # 页面A的功能基线
├── pageA_from_pageB.feature  # 从B切换来后，页面A的功能回归
└── pageA_from_pageC.feature  # 如果C切换过来也有问题，同理
```

最低侵入性方案：**Tag + Hook**
```gherkin
# pageA.feature 原封不动，只加一个 Tag
@from_pageB
Feature: 页面A - 从页面B切换过来后的功能回归

  Scenario: 查询功能正常
    Given 用户打开"页面A"
    When ...
    Then ...

  Scenario: 筛选功能正常
    Given 用户打开"页面A"
    When ...
    Then ...
```
Hook文件
```javascript
// hooks.js
Before({ tags: '@from_pageB' }, async () => {
  await navigateTo('页面A');
  await navigateTo('页面B');
  await navigateTo('页面A');  // 模拟切换路径
});
```
